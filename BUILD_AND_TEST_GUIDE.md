# Build & Test Guide for Woosh Integration\n\n## Prerequisites\n\n- Android Studio installed\n- Laravel installed with PHP 8.0+\n- MySQL running\n- Firebase project configured\n- Emulator or physical Android device\n\n---\n\n## Part 1: Build Android App\n\n### Step 1: Clean Build\n\n```bash\n# In Android Studio Terminal or Command Prompt\ncd c:\\Users\\Thinkpad\\AndroidStudioProjects\\Woosh\n\n# Clean previous build\n.\\gradlew clean\n\n# Build debug APK\n.\\gradlew assembleDebug\n```\n\n**Expected Output**:\n```\nBUILD SUCCESSFUL in XXs\n```\n\n**If Build Fails**:\n- Check for Kotlin compilation errors: `getDiagnostics` should show no errors\n- Ensure all dependencies are downloaded: `./gradlew build --refresh-dependencies`\n- Check Android SDK version in `build.gradle.kts`\n\n### Step 2: Install on Device/Emulator\n\n```bash\n# Option A: Using Android Studio\n# 1. Click \"Run\" → \"Run 'app'\"\n# 2. Select device/emulator\n# 3. Click \"OK\"\n\n# Option B: Using Command Line\n.\\gradlew installDebug\n```\n\n**Expected Output**:\n```\nInstalling APK 'app-debug.apk' on 'emulator-5554'\nInstalled successfully\n```\n\n### Step 3: Verify App Launches\n\n1. Open Android emulator or device\n2. Look for \"Woosh\" app icon\n3. Tap to launch\n4. Should see login/register screen\n\n---\n\n## Part 2: Setup Laravel Server\n\n### Step 1: Install Dependencies\n\n```bash\ncd c:\\Users\\Thinkpad\\AndroidStudioProjects\\Woosh\\admin-woosh\n\n# Install PHP dependencies\ncomposer install\n\n# If composer not found, use:\nphp -r \"copy('https://getcomposer.org/installer', 'composer-setup.php');\"\nphp composer-setup.php\nphp composer.phar install\n```\n\n### Step 2: Configure Environment\n\n```bash\n# Copy .env.example to .env\ncopy .env.example .env\n\n# Generate app key\nphp artisan key:generate\n```\n\n**Edit `.env` file**:\n```env\nAPP_NAME=Woosh\nAPP_ENV=local\nAPP_DEBUG=true\nAPP_URL=http://localhost:8000\n\nDB_CONNECTION=mysql\nDB_HOST=127.0.0.1\nDB_PORT=3306\nDB_DATABASE=woosh_db\nDB_USERNAME=root\nDB_PASSWORD=\n\nFIREBASE_CREDENTIALS=/path/to/firebase-credentials.json\n```\n\n### Step 3: Setup Database\n\n```bash\n# Create database\nmysql -u root -e \"CREATE DATABASE woosh_db;\"\n\n# Run migrations\nphp artisan migrate\n\n# Seed data (optional)\nphp artisan db:seed\n```\n\n### Step 4: Start Laravel Server\n\n```bash\n# Terminal 1: Start Laravel server\nphp artisan serve --host=0.0.0.0 --port=8000\n```\n\n**Expected Output**:\n```\nLaravel development server started: http://0.0.0.0:8000\n```\n\n**Verify Server is Running**:\n- Open browser: `http://localhost:8000`\n- Should see Laravel welcome page or redirect to admin dashboard\n\n---\n\n## Part 3: Configure Mobile App\n\n### Step 1: Set Server IP\n\n1. Verify Laravel server is running: `php artisan serve --host=0.0.0.0 --port=8000`\n2. Check server IP in mobile app settings\n3. Verify firewall allows port 8000\n4. Try pinging server: `ping 10.10.45.41` (replace with your IP)\n\n### Issue: \"Jadwal tidak muncul\" (Trips not appearing)\n\n**Solution**:\n1. Admin: Click \"Sync Firebase\" button\n2. Wait 5 seconds\n3. Mobile: Refresh trips screen\n4. Check MySQL: `SELECT * FROM trips;`\n\n### Issue: \"Tiket tidak muncul di admin\" (Ticket not in admin)\n\n**Solution**:\n1. Verify user registered (check MySQL: `SELECT * FROM users;`)\n2. Verify trip exists (check MySQL: `SELECT * FROM trips;`)\n3. Check Laravel logs: `php artisan logs`\n4. Check Firestore: verify ticket document exists\n\n### Issue: \"Refund tidak berubah\" (Refund status not changing)\n\n**Solution**:\n1. Verify ticket_code matches in Firestore and MySQL\n2. Check user email is same in both\n3. Check MySQL: `SELECT * FROM tickets WHERE ticket_code = 'WSH-TK-xxx';`\n4. Check Laravel logs for errors\n\n### Issue: \"sync-firebase returns 404\"\n\n**Solution**:\n1. Clear route cache: `php artisan route:clear`\n2. Verify admin user role: `SELECT * FROM users WHERE email = 'admin@example.com';`\n3. Verify logged in as admin\n4. Check middleware in `routes/web.php`\n\n### Issue: \"Broadcast notification tidak muncul\" (Notification not appearing)\n\n**Solution**:\n1. Verify Firestore rules allow read/write\n2. Check user has firebase_uid: `SELECT * FROM users WHERE firebase_uid IS NOT NULL;`\n3. Check Firestore: `/users/{uid}/notifications` collection\n4. Check NotificationScreen reads from Firestore\n\n---\n\n## Part 6: Performance Optimization\n\n### Mobile App\n- Rebuild only when Kotlin code changes\n- Use emulator for faster testing\n- Clear app cache if issues persist: Settings → Apps → Woosh → Storage → Clear Cache\n\n### Laravel Server\n- Use `php artisan optimize` for production\n- Enable query caching for better performance\n- Monitor logs: `tail -f storage/logs/laravel.log`\n\n### Database\n- Add indexes on frequently queried columns\n- Backup before major changes\n- Monitor query performance\n\n---\n\n## Part 7: Deployment\n\n### Before Going Live\n\n1. **Security**\n   - [ ] Set `APP_ENV=production` in `.env`\n   - [ ] Set `APP_DEBUG=false` in `.env`\n   - [ ] Update Firebase Firestore rules to production\n   - [ ] Use HTTPS for API calls\n\n2. **Database**\n   - [ ] Backup MySQL database\n   - [ ] Backup Firestore data\n   - [ ] Run migrations on production\n\n3. **Testing**\n   - [ ] Run all 8 test scenarios\n   - [ ] Test on multiple devices\n   - [ ] Test with slow network\n   - [ ] Test with offline mode\n\n4. **Monitoring**\n   - [ ] Setup error logging\n   - [ ] Monitor server performance\n   - [ ] Monitor Firebase usage\n   - [ ] Setup alerts for errors\n\n---\n\n## Summary\n\n**Total Setup Time**: ~30 minutes\n**Total Testing Time**: ~20 minutes\n**Total**: ~50 minutes\n\nAfter completing all steps and tests, the system is ready for production deployment.\n\n---\n\n**Last Updated**: May 26, 2026\n**Status**: ✅ Ready for Testing\n
